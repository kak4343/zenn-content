## なぜ「朝」なのか

朝の 30 分は、人生で最も希少な可処分時間です。研修医 1 年目で USMLE 受験を始めたとき、最初に取り組んだのは「朝 5 時起き」ではなく「朝 7 時に起きた瞬間、何をすべきかが既に Notion に並んでいる状態を作ること」でした。決断疲れを減らすほうが、勉強時間を増やすより費用対効果が高かったからです。

scheduled-tasks は、この「決断疲れ削減」を Claude Code 側に肩代わりさせる仕組みです。本章では、私が実際に毎朝 6 時 / 7 時に回しているジョブ群と、その設計判断を公開します。

---

## 私の朝のジョブ群(2026 年 5 月時点)

```
05:00  email-cleanup スマートフォルダ更新(月初 1 日のみ)
06:00  content-publishing-collect (HN + dev.to + Anthropic News → Notion)
07:00  ophthalmology-weekly-review (月曜のみ、PubMed 巡回 → Notion 投入)
07:00  trend-research(月初 1 日のみ、ファッション / 旅行 / グルメ)
07:30  (起床時刻)
07:35  Notion を開く、夜間に届いた結果を 5 分で目視レビュー
```

合計 5 ジョブ。本人不在で動きます。起床時の Notion には、その日に目を通すべき情報が既に整理された状態で並んでいます。

---

## ジョブごとの設計判断

### content-publishing-collect(毎朝 6:00)

目的: α 発信のネタを毎朝補充する。
ソース: Hacker News + dev.to + Anthropic 公式 News。
出力: Notion Knowledge DB に Auto-imported チェック付きで投入。
重複検出: タイトルを英数字以外を削った正規化文字列で SHA-256 ハッシュ、同一バッチ内では除外。Notion 側の重複は週次サマリで気づけば手動で削除。
失敗時の挙動: exit code 1 で終了、PushNotification を送信。スクリプト全体はベストエフォート(1 ソース失敗しても他は試行)。

### ophthalmology-weekly-review(毎週月曜 7:00)

目的: 1 週間ぶんの主要眼科ジャーナルと PubMed から注目論文を 5-7 本選び、日本語要約を Notion に投入。
ソース: Ophthalmology / JAMA Ophthalmology / IOVS / AJO / BJO / Retina / Cornea / Journal of Glaucoma + PubMed。
除外: predatory journals、medRxiv プレプリント。
コスト: 週 0.08 ドル前後(Anthropic API 呼び出し費用)。
失敗時: PushNotification で通知、当日手動再実行可。

### email-cleanup(月初 1 日 10:00、launchd 経由)

目的: 個人メール 3 アカウント(iCloud / Yahoo / Gmail)を AppleScript で整理。
方針: パターン認識による自動振り分け、フラグ付きメールは絶対消さない、ゴミ箱への move のみ(永久削除しない、ゴミ箱は 30 日で自動消去)。
出力: `~/Documents/Email_Cleanup_Reports/report_YYYY-MM.txt`。

### trend-research(月初 1 日 7:00)

目的: ファッション / 旅行 / グルメの月次トレンドを 10 ソースから集約。
出力: `~/Documents/trend-research/YYYY-MM_report.md` → Notion Inbox に push。
非依存: Instagram には依存しない(API 制約)。代わりに PR TIMES / WWD JAPAN / 食べログ / トリップアドバイザー / じゃらん等の Web 公開ソース。

## launchd vs scheduled-tasks の使い分け

macOS には 2 つのスケジューラがあります。どちらを使うかの判断基準を書きます。

- **launchd を使う**: 
  - macOS ネイティブのコマンド(AppleScript、osascript、Bash スクリプト)を回したい
  - Claude を呼ばない(API コストをかけない)
  - 例: `email-cleanup` の AppleScript
- **scheduled-tasks を使う**:
  - Claude を呼び、AI 的な要約・分類・判断を含む
  - 結果を Notion DB や PushNotification に流したい
  - 例: `ophthalmology-weekly-review`, `content-publishing-collect`

両方の使い分けは「AI が必要か」で決めます。AI 不要の整理作業は launchd、AI が要約 / 分類する作業は scheduled-tasks。

## 章末演習

自分の朝 30 分を AI に肩代わりさせる設計を書いてください。次の 3 ステップで進めます。

**ステップ 1**: 起床時に「目を通すべき情報」を 5 個書き出す。
例: 業界ニュース、SNS のメンション、メール、論文、家族からのメッセージ

**ステップ 2**: それぞれに次の 3 つを割り当てる:
- 朝に必要 / 朝じゃなくていい
- 自動収集できる / 人間判断が要る
- Notion に届けば良い / 通知も必要

**ステップ 3**: 「朝に必要 + 自動収集できる」項目を scheduled-tasks に乗せる。最初は 1 ジョブだけ動かして、1 週間運用してから 2 個目を足してください。

私の経験では、1 つ目のジョブを 2 週間運用して「これは無いと困る」と感じた瞬間から、3 つ目以降を一気に組めるようになります。
