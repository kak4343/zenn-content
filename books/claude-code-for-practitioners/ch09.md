## 翻訳ではなく要約である理由

英語論文を読むときの最も非効率な選択は「全文を翻訳してから読む」ことです。全文翻訳は時間がかかり、しかも翻訳された日本語を読む時点で元の英語の精度を失います。

私が運用している方針は次の通りです。

- **元の英語のままアブストラクトと結論を読む**(原文の精度を保つ)
- **要約のみ日本語で書く**(高速スキャンと記憶の補助)
- **専門用語は英語のまま残す**(ブレを防ぐ)

つまり「翻訳」ではなく「日本語要約」を目的としています。Claude にもこの方針を明示的にプロンプトで指示します。

## 3 段プロンプト構造

私の `ophthalmology-weekly-review` skill は、論文 1 本につき 3 段のプロンプトで要約します。各段で AI が出すべき情報を絞り、ブレを抑えています。

### 第 1 段: メタ情報の構造化

```
入力: 論文の bibliographic info(タイトル / 著者 / ジャーナル / 年 / DOI)
出力: 構造化された JSON(title / title_ja / authors / journal / year / doi / pmid / specialty / design)
```

ここでは要約しません。情報の正規化だけです。AI が `design` を推定するときに失敗しやすいので、プロンプトに次の選択肢を明示します。

```
"design" は次のいずれかから選ぶ:
- RCT(randomized controlled trial)
- Observational(cohort / case-control / cross-sectional)
- Meta-analysis / Systematic review
- Case series
- Basic science(動物 / 細胞 / 分子)
- Review article(narrative review)
```

選択肢を制限することで、ハルシネーションが減ります。

### 第 2 段: 要約生成

```
入力: 第 1 段の構造化情報 + アブストラクト本文
出力: 200-400 字の日本語要約(主要結果と数値、統計的有意性、限界)
```

プロンプトに含める制約:

- 数値は原文のまま転記すること(換算しない)
- p 値、信頼区間、絶対リスク減少 などの統計値があれば必ず含める
- 限界(limitations)を 1 文で書く

### 第 3 段: 臨床的意義の判定

```
入力: 第 1 段 + 第 2 段の出力
出力: 1-2 文の「臨床的意義」セクション
```

ここが最もハルシネーションしやすい部分です。プロンプトに次を明示しています。

```
臨床判断を変える内容なら、何をどう変えるかを 1 文で書く。
変えない場合は「現行の管理を変更する根拠は不十分」と書く。
美辞麗句は禁止。「画期的」「重要」などの形容詞は使わない。
```

特に最後の「形容詞禁止」は AI が好む表現を直接ブロックします。

## Glossary の設計

医学英語は同義語が多く、Claude に任せると訳語がブレます。たとえば:

- `macular degeneration` → 黄斑変性 / 黄斑変性症 / 加齢黄斑変性
- `hypertension` → 高血圧 / 高血圧症 / 本態性高血圧

これを統一するため、私の skill には小さな glossary を持たせています。

```python
GLOSSARY = {
    "age-related macular degeneration": "加齢黄斑変性",
    "AMD": "AMD(加齢黄斑変性)",
    "diabetic retinopathy": "糖尿病網膜症",
    "DR": "DR(糖尿病網膜症)",
    "intraocular pressure": "眼圧",
    "IOP": "眼圧",
    "optical coherence tomography": "光干渉断層計",
    "OCT": "OCT(光干渉断層計)",
    # ... 50 用語
}
```

50 用語あれば、自分の専門領域の論文要約は 9 割カバーできます。glossary は SKILL.md の references/ 配下に Markdown で置き、prompt の system 部に注入します。

## 英語残しのルール

要約の中で、次のものは **英語のまま** 残します。

- 略語(AMD / IOP / OCT / RCT 等)
- 解剖学用語(macula / retina / cornea 等の専門領域内で頻出のもの)
- 統計指標(p value / 95% CI / HR / OR / NNT 等)
- ドラッグ名(分子名は英語、商品名は併記)

逆に、次は **日本語に訳します**:

- 臨床判断の方針(「経過観察」「治療開始」「専門医紹介」)
- 患者プロファイル(「65 歳男性」「2 型糖尿病既往」)
- 主要結果の説明文(「視力が 0.3 改善した」)

このハイブリッドが、日本人医師にとって最も読みやすい形です。完全日本語化すると専門用語のブレが出る、完全英語化すると 30 秒スキャンには向かない、その中間を取ります。

## 失敗パターン

運用していて気づいた失敗パターンを 3 つ書きます。

**失敗 1: アブストラクトだけで要約させる**

Claude にアブストラクトを渡しただけだと、本文の「Limitations」セクションを見ない要約になります。アブストラクトには limitations が書かれていないことが多いからです。

対策: skill 側で論文 PDF の Limitations セクション or Discussion 末尾を別途取得して、要約プロンプトに同梱する。

**失敗 2: glossary を渡しても訳語がブレる**

長い glossary をプロンプトに入れても、たまにブレます。

対策: 要約生成後に、glossary の英語側キーで grep して、対応する日本語が出ているか後処理でチェック。違反があれば再生成。

**失敗 3: 統計的有意性の取り違え**

p < 0.05 と p > 0.05 を AI が逆に書いてしまう事故が、月 1 回くらい起きます。

対策: 要約に p 値が含まれる場合、人間が必ず最終確認する。skill の SKILL.md に「数値は出典で必ず確認」と明記。

## 章末演習

自分の専門領域で glossary を 30 用語作ってください。次の手順:

**ステップ 1**: 直近 3 ヶ月の PubMed 検索で、自分の専門領域の論文を 20 本リストアップ。

**ステップ 2**: 各論文のアブストラクトから、頻出する英語専門用語を抽出。

**ステップ 3**: 同義語を統合し、訳語を 1 つに決める。略語と並列で表記する。

**ステップ 4**: Markdown で 30 用語の表を作り、自分の skill の references/ 配下に置く。

これで他科に移植可能な要約 skill の最小構成が手に入ります。
