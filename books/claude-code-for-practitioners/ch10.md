## この章の前提

本章は「副業 = 短期労働を増やす」という発想を取りません。FIRE 志向で、ストック型・資本収益型を優先する読者を想定します。具体的には:

- 投資ノートの構造化と意思決定支援
- 不動産物件のスクリーニング自動化
- 税務サポート(節税策の整理、医療費控除、ふるさと納税)
- アフィリエイトサイトの裏方自動化

いずれも本章で扱う「副業収益」の意味は **時間を売る側ではなく、システムを作る側** です。Claude Code は前者を効率化するためのものではなく、後者の建設コストを下げるためのものです。

---

## パターン 1: 投資ノートの構造化

私の投資判断は次の構造で記録しています。

- 銘柄(または物件)名
- 判断日 / 判断時の数値(株価、利回り、PER、その他)
- 仮説(なぜ買う / 買わないと判断したか、3 行以内)
- 撤退ライン(何が起きたら売る or 損切るか)
- 1 ヶ月後 / 3 ヶ月後 / 1 年後のレビュー欄

これを Markdown で書いて、`~/invest/decisions/YYYY-MM-DD-ticker.md` のように保存しています。

Claude Code に任せている仕事は次の通り:

- 月次の **未レビュー判定** 抽出: ファイル一覧から、最終更新が 1 ヶ月以上前のものを列挙
- 仮説の **反駁的レビュー**: 「この仮説を否定する観点を 3 つ挙げてください」と Claude に投げる
- 撤退ラインの **自動チェック**(株価系): 当日の価格を取得し、撤退ラインを超えていれば通知

注意: 投資判断そのものは AI に任せません。AI は仮説の盲点を指摘する役、損益のチェッカー、レビュー漏れの監視役です。判断は人間に残します。

## パターン 2: 不動産物件のスクリーニング

第 4 章の「skill 駆動 vs cron 駆動」のハイブリッド設計の典型例です。

私の運用ルール:

- 週 1 回、scheduled-tasks で 4 つの不動産情報サイトを巡回
- 6 つの数値基準(表面利回り 7% 以上、築年数 35 年以内、駅徒歩 10 分以内、修繕履歴あり、与信枠内の価格帯、構造が RC または木造)で機械的に絞り込み
- 1 つでも基準を割ったら除外
- 全項目通過した物件のみ、Notion に投入

月 30 件 → 3-5 件に絞れます。30 秒の目視レビューで現地訪問する 1-2 件まで落ちます。

医師ローン枠を活用する人にとって、**スクリーニングの自動化は最大の防御策** です。営業電話の多さに比例して判断疲れが進み、悪い物件を勢いで買う事故が起きます。事故を防ぐ最善策は、人間が判断する前段に冷たいフィルタを置くこと。

## パターン 3: 税務サポート

医師の副業所得は、確定申告の構造が複雑になりがちです。Claude Code で次を自動化しています。

- 経費計上候補のメール抽出(交通費、書籍、学会費、研究関連)
- ふるさと納税の上限額シミュレーション(年収、家族構成、控除を入力すると上限額を出す)
- 医療費控除の領収書整理(写真 → OCR → 月別集計)
- 開業医・副業所得者向けの節税策チェックリスト(iDeCo、小規模企業共済、青色申告)

注意: 税務判断そのものは税理士に任せます。Claude Code は素材整理係です。私は会計事務所への支払い時間を圧縮するのが目的で、自分で確定申告を完結させるのが目的ではありません。

---

## パターン 4: アフィリエイトサイトの裏方

私が運営している `駆け出しドクター ポイ活ラボ`(医師向けポイ活紹介サイト)では、Claude Code の役割は次の通り。

- 週次で、医療系ポイントサイトの高単価案件を巡回
- 案件のサマリを Notion に投入
- Astro + Cloudflare Pages の静的サイトを自動再ビルド
- 法令遵守(ステマ規制対応、PR 表記の自動挿入)

患者情報を扱わない、第三者の公開済み情報のみを扱う設計で、医療×副業の組み合わせのなかでは安全性が高いカテゴリです。

---

## やってはいけないこと

ここまでの 4 パターンが共通して避けているのは:

- **患者情報を含む業務を副業化しない**: 医療翻訳・医療文書添削・症例コンサル等。報酬は良くても法的・倫理的リスクが大きすぎる
- **本業の評判で売る副業をしない**: 自分の所属病院 / 専門領域を看板にしたコンサルは、トラブル時の損害が本業に波及する
- **時間を売る型の副業を Claude Code で効率化しようとしない**: 時間給型は効率化のメリットが薄く、ストック型に移行する方が本筋

## 章末演習

自分の副業候補を 3 つ書き出し、次の 3 つの問いに答えてください。

- ストック型か、フロー(時間給)型か
- 患者情報・症例情報を扱う必要があるか
- 本業のリスクに波及する可能性があるか

ストック型 + 患者情報なし + 本業リスクなし に当てはまる候補だけを残してください。残ったものから、最初の 1 つを Claude Code で支える skill 設計を試作します。
