## 本書のしめくくり

ここまでの 11 章は、私が個人インフラとして Claude Code を運用するうえでの設計判断を書いてきました。最終章では、それらを商品化する判断について書きます。

商品化を考える理由は人によって違います。FIRE 志向で資本収益を増やしたい(私の場合)、専門知識を公的に共有したい、研究費の足しにしたい、いろいろです。本章で扱うのは、商品化するかどうかの判断軸と、商品化する場合の最初の 90 日の動かし方です。

---

## 商品化の 4 つの選択肢

個人 skill を商品化するパスは、現実的には 4 つに収束します。

### 選択肢 A: Zenn Book / 電子書籍(印税ストック型)

- 強み: 一度書けば永続的に売れる、英語版併出でグローバル展開可能
- 弱み: 執筆工数が大きい(章ごとに 2-3 時間)
- 適しているケース: 文章で完結する設計論や思想

### 選択肢 B: Udemy / オンライン講座(動画 + ストック)

- 強み: 単価が高い(1 講座 1-3 万円)、Udemy のオーガニック流入が大きい
- 弱み: 動画撮影・編集に時間がかかる、英語講座は競合が多い
- 適しているケース: 手を動かす実装を見せたいもの

### 選択肢 C: note / Substack 有料記事(段階値上げ型)

- 強み: 早期から収益化可能、コミュニティ構築と組み合わせやすい
- 弱み: 単体価格に上限がある、note は日本限定
- 適しているケース: 特定の読者ペルソナに対する深い実用書

### 選択肢 D: SaaS / GPT Store / カスタムアプリ

- 強み: 月額課金が積み上がる、技術的レバレッジが大きい
- 弱み: 顧客獲得が難しい、運用責任が重い
- 適しているケース: 自分用ツールを汎用化できる場合

私の α 立ち上げ(本書を含む)では、B と C と A を組み合わせ、D は P1 以降に検討する設計にしました。

---

## 最初の 90 日(P0)の動かし方

私の場合の P0(2026-05-30 〜 2026-08-31)はこう動いています。

- 第 1 週: アカウント整備、自己紹介投稿、最初の skill 公開
- 第 2-4 週: 週次の発信ルーチン確立、反応の観察、ペルソナ修正
- 第 5-8 週: Substack 第 1 弾の公開、note 商品の第 1 章公開(β 価格)
- 第 9-12 週: Zenn Book 主要章公開、フォロワー実数で Go/No-Go 判定

KPI は次の 4 つに絞っています。

- 英語 X フォロワー 500
- 日本語 X フォロワー 1,000
- Substack 購読者 100
- Zenn Book 6 章完成

達成数 = 4 なら次のフェーズ(英語 Udemy 講座、Substack 有料化)へ進む。達成数 0-1 なら戦略を縮小して日本語専念に切り替える。

## 商品化しない選択

ここを書かないと不誠実です。商品化しない選択肢も真剣に検討しました。

理由は次の通り:

- 副業所得が増えると確定申告が複雑化する
- 顧客対応(問い合わせ・返金)の運用コストが本業を圧迫する可能性
- 失敗(売れない)時の心理的負担

私が商品化に踏み切ったのは、本業外の時間に資本収益を積み上げる戦略上の必要性と、Claude Code 関連の発信が「医師 × エンジニア」というニッチで他者の参入が薄い時期だったからです。逆に言えば、この 2 つが満たされない人には商品化を急ぐ理由はないかもしれません。

## 章末演習(最終)

これまでに自分が作った skill / hooks / scheduled-tasks のリストから、次の 3 つに振り分けてください。

- **商品化候補**: 他人の役に立つ、再現可能、機密データを含まない
- **発信候補(無料)**: 商品にはならないが、共有して評判につながる
- **内部用**: 自分の業務専用、公開しない

商品化候補から最も再現性の高い 1 つを選び、選択肢 A-D のどれで商品化するかを決めてください。決まったら、最初の 90 日のマイルストーンを 4 つ書き出します。
